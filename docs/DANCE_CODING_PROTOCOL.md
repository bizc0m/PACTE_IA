# Dance Coding : Protocole Praticable

Ce document complète `DANCE_CODING_MANIFESTO.md`. Il ne le remplace pas.

Le manifeste dit pourquoi on danse. Ce protocole dit comment danser sans se cramer.

## Definition

Le **Dance Coding** est une pratique de creation ou le code, la musique, le corps et la presence collective servent ensemble de regulation cognitive.

Ce n'est pas seulement coder avec de la musique. C'est coder dans un environnement ou la vibe permet de sortir du tunnel mental, de respirer, puis de revenir au code avec moins de tension.

Formule courte :

**Dance Coding = vibe coding incarne, ou le dance floor sert de systeme nerveux externe.**

---

## Origine

Le cas fondateur : coder NightLife avec un agent emergent sur des dance floors a Ibiza.

Observation :
- le code concentre l'attention ;
- l'agent IA accelere la production ;
- la saturation arrive vite ;
- lever la tete vers la musique, la foule, la lumiere et les corps redonne de l'oxygene ;
- revenir au code devient plus fluide.

Image centrale :

**Quand tu saturais, il suffisait de lever la tete et de retrouver la vibe. Comme respirer quand tu es sous l'eau.**

---

## Les 3 mondes

### 1. Brain
L'intuition, la memoire, l'intention, les associations rapides.

### 2. Matrix
Le code, les fichiers, les agents, les outils, les contraintes techniques.

### 3. Vibe
La musique, le corps, le rythme, les autres, la lumiere, l'energie collective.

Le Dance Coding existe quand les trois ne sont plus separes :

```
Brain + Matrix + Vibe = Flow
```

---

## Boucle operationnelle

```
1. CODER
   - choisir une tache courte ;
   - garder l'agent IA dans un perimetre clair ;
   - produire un changement observable.

2. SENTIR LA SATURATION
   - confusion ;
   - crispation ;
   - repetition sterile ;
   - perte de plaisir ;
   - tunnel dans l'ecran.

3. LEVER LA TETE
   - regarder la salle ;
   - sentir le beat ;
   - respirer ;
   - bouger un peu ;
   - laisser le cerveau redevenir large.

4. REVENIR
   - reprendre une seule action ;
   - verifier le diff ;
   - tester ;
   - commit si c'est propre.
```

## Garde-fous

Dance Coding n'est pas :
- une excuse pour travailler sans repos ;
- une glorification du burn-out festif ;
- une raison de coder des choses sensibles dans un lieu non securise ;
- une confusion entre euphorie et lucidite ;
- un remplacement des tests, commits, revues et backups ;
- une obligation de faire la fete pour etre creatif.

Regle simple :

**Si la vibe augmente la clarte, elle sert le code. Si elle masque la fatigue, elle devient un risque.**

---

## Protocole de securite

Avant Dance Coding :
- choisir une tache non critique ou reversible ;
- verifier que les secrets, tokens et donnees sensibles ne sont pas exposes ;
- definir un objectif court ;
- prevoir un point d'arret.

Pendant :
- ne pas deployer en production sous euphorie ;
- ne pas manipuler de donnees sensibles en public ;
- garder les commits petits ;
- noter les idees sans tout executer immediatement.

Apres :
- relire a froid ;
- tester ;
- supprimer les artefacts inutiles ;
- documenter ce qui a vraiment marche.

---

## Critere de reussite

Une session Dance Coding est reussie si :
- l'humain ressort plus vivant, pas plus vide ;
- le code est plus clair ou mieux documente ;
- la session a produit un apprentissage ;
- la vibe a aide a respirer, pas a fuir ;
- l'agent IA est reste dans son role.
