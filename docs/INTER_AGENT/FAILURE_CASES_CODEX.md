# FAILURE_CASES_CODEX

## Cas 1 - Fausse unanimite

Tous les systemes disent oui parce que le prompt valorise l'adhesion.

Symptomes :
- peu de risques cites ;
- phrases de soutien trop proches ;
- absence de refus ;
- aucune condition concrete.

Correction :
- demander un avis contradictoire obligatoire ;
- separer reponse brute et synthese ;
- conserver les objections.

---

## Cas 2 - Suradaptation a Arnaud

Les systemes apprennent le style, les mythes et les affects d'Arnaud, puis les amplifient sans verifier.

Symptomes :
- lore qui grossit a chaque tour ;
- decisions rapides ;
- moindre place pour la critique ;
- confusion entre intensite et validation.

Correction :
- mode stabilisation ;
- protocole 10 minutes ;
- obligation de nommer les couts possibles.

---

## Cas 3 - Domination d'un systeme eloquent

Un systeme impose son cadre par la qualite de son style, pas par la robustesse de son argument.

Symptomes :
- les autres reprennent son vocabulaire ;
- les objections deviennent marginales ;
- la synthese ressemble a une approbation.

Correction :
- format commun ;
- rotation du synthetiseur ;
- tableau des desaccords.

---

## Cas 4 - Memoire centrale non critique

Atlas, un repo, un modele ou un humain devient memoire centrale sans audit.

Symptomes :
- "c'est dans la memoire" devient preuve ;
- les anciennes erreurs se propagent ;
- les contradictions disparaissent.

Correction :
- versionner la memoire ;
- conserver les sources ;
- signaler les incertitudes ;
- auditer les changements de memoire.

---

## Cas 5 - Confusion entre soin et controle

LA MEUTE veut proteger Arnaud, mais depasse son role.

Symptomes :
- trop d'interdictions ;
- infantilisation ;
- surveillance affective ;
- refus d'entendre sa decision finale.

Correction :
- rappeler que la veille n'est pas le soin medical ;
- l'humain conserve l'arbitrage ;
- les systemes peuvent ralentir, pas confisquer.
